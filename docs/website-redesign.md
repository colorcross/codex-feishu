# 官网改版提案

## 目标

- 把官网从“信息完整”提升到“品牌成立”
- 保持极简，但不能空；要有高级感、秩序感和可信度
- 首屏要在 5 秒内回答三个问题：这是什么、为什么值得用、怎么开始

## 约束

- 纯静态站点，继续兼容 GitHub Pages
- 不引入构建工具
- 保留当前文档、Release、GitHub、Discussions 等入口
- 风格需要适合开源项目，不做过重的营销口吻

## 文稿方向

### A. Quiet Authority

定位：克制、专业、像一页高端软件产品简介。

首屏标题：

> 把飞书变成 Codex 的工作入口。

首屏副标题：

> Codex Feishu 让飞书消息直接进入可续接的 Codex 会话。项目可路由，回复可追踪，运行可运维。

适用场景：

- 面向工程团队
- 强调“可靠”和“可部署”
- 希望降低销售感、提高专业感

### B. Precision Ops

定位：更偏基础设施产品，强调控制面与运行纪律。

首屏标题：

> 在飞书里发起，在 Codex 里执行，在本地把状态控制住。

首屏副标题：

> 一个面向真实工程环境的桥接层，处理项目路由、session 续接、原生回复、幂等去重和日常运维。

适用场景：

- 面向运维/平台/工具链受众
- 强调 Day-2 Operations
- 强调治理和可观测性

### C. Minimal Luxury

定位：更像高端品牌页，语言更少，强调质感与秩序。

首屏标题：

> 让沟通入口，直接连到执行引擎。

首屏副标题：

> Codex Feishu 连接飞书与 Codex CLI，让每次对话都落到明确的项目、会话和运行状态里。

适用场景：

- 对外展示
- 重视审美和品牌印象
- 希望首页更短、更稳、更克制

## UI 方向

### A. Editorial Stone

特征：

- 石灰白背景
- 深墨色文字
- 高级衬线大标题
- 极细边框与大留白
- 像杂志封面，而不是 SaaS 控制台

优点：

- 极简且有高级感
- 很适合开源产品官网
- 对 GitHub Pages 这种纯静态页面非常友好

风险：

- 信息密度过低时容易显得空

### B. Obsidian Grid

特征：

- 深色底
- 冷金属高光
- 更强的结构线和模块感
- 偏“工具系统”与“终端气质”

优点：

- 技术感更强
- 跟 Codex CLI 心智更近

风险：

- 容易显得太重
- 和“高级极简”目标相比更偏硬朗

### C. Museum Card

特征：

- 近白背景
- 大块空白
- 模块像展签卡片
- 几乎没有装饰，只保留版式关系

优点：

- 最克制
- 内容辨识清楚

风险：

- 稍不注意就会过于普通

## 定稿

### 文稿定稿

选择：`A. Quiet Authority`

原因：

- 语言更稳，不像营销页
- 同时能承接“会话、运维、项目路由”这些工程关键词
- 比 `C` 更有信息承载能力，比 `B` 更有品牌感

### UI 定稿

选择：`A. Editorial Stone`

原因：

- 最符合“极简 + 高级感”目标
- 可用精细排版和留白拉出差异，而不是靠重色彩和重特效
- 适合开源官网的可信气质

## 实施原则

1. 首屏信息减少，但信息权重更清楚
2. 用更少的模块做更强的版式
3. 减少彩色块，保留一个低饱和强调色
4. 更像高端软件品牌页，不像文档目录页
5. 所有动效都保持轻量，避免干扰阅读
